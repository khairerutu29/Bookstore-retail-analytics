# Bookstore Sales & Customer Analytics (SQL Project)

## Project Overview
This project analyzes the operations of a bookstore using SQL.
The database contains information about books, customers, orders, and marketing spending.
The goal of this project is to demonstrate SQL skills used by data analysts to extract business insights such as revenue generation, customer behavior, marketing profitability, and sales trends.
## Database Structure
The database contains four main tables:

### 1. Books
Stores information about books available in the store.
Columns:
* book_id (Primary Key)
* title
* genre
* price
* stock
### 2. Customers
Stores customer details.
Columns:
* customer_id (Primary Key)
* name
* city
* signup_date
### 3. Orders
Stores purchase transactions made by customers.
Columns:
* order_id (Primary Key)
* customer_id (Foreign Key)
* book_id (Foreign Key)
* quantity
* order_date
### 4. MarketingSpend
Stores marketing costs associated with customers.
Columns:
* spend_id (Primary Key)
* customer_id (Foreign Key)
* spend_amount
## Key SQL Analysis Performed
The project includes multiple SQL queries to answer business questions:
• Total revenue generated by each book
• Books with low inventory stock
• Customer RFM analysis (Recency, Frequency, Monetary)
• Profit analysis comparing marketing spend and revenue
• Monthly revenue trends
• Repeat customers analysis
• Average order value calculation
• Frequently bought together books
• Customer inactivity analysis

## Example Query

```sql
SELECT 
    b.title,
    SUM(o.quantity) AS total_units_sold,
    SUM(o.quantity * b.price) AS total_revenue
FROM Orders o
JOIN Books b ON o.book_id = b.book_id
GROUP BY b.title
ORDER BY total_revenue DESC;
```
This query calculates the total units sold and revenue generated for each book.
